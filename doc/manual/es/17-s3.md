# S3

## Requisitos previos

En primer lugar, necesitas tener una cuenta de AWS, un usuario IAM y un bucket de S3.

Para crear un usuario IAM:

1. Inicia sesión en la Consola de administración de AWS y abre la [consola de IAM][aws_iam].

2. En el panel de navegación, elige Usuarios y después elige Agregar usuarios.

3. Escribe el nombre de usuario para el nuevo usuario.

4. Selecciona el tipo de acceso para que sea tanto acceso programático como acceso a la Consola de administración de AWS.

5. Elige Siguiente: Permisos.

6. En la página Establecer permisos, selecciona adjuntar políticas existentes directamente, busca AmazonS3FullAccess y elige esa opción, después elige Siguiente: Revisar, después elige Agregar permisos.

7. Elige Siguiente: Etiquetas.

8. Elige Siguiente: Revisar para ver todas las opciones que realizaste hasta este punto. Cuando estés listo para continuar, elige Crear usuario.

9. Para ver las claves de acceso de los usuarios (IDs de clave de acceso y claves de acceso secretas), elige Mostrar next a cada contraseña y clave de acceso que desees ver. Para guardar las claves de acceso, elige Descargar .csv y procede a guardar el archivo en una ubicación segura.

Ahora estás listo para crear un bucket de S3. Para crear un bucket de S3:

1. Inicia sesión en la Consola de administración de AWS usando las credenciales de tu usuario IAM y abre la [consola de Amazon S3][aws_s3].

2. Elige Crear bucket.

3. En Nombre del bucket, ingresa un nombre para tu bucket.

4. En Región, elige la región de AWS donde deseas que resida el bucket.

5. Mantén los valores predeterminados tal como están y elige Crear bucket.

## Modificar la configuración de pgmoneta

Necesitas tener espacio de almacenamiento para 1 backup en tu computadora local.

Cambia `pgmoneta.conf` para agregar

``` ini
storage_engine = s3
s3_region = the-aws-region
s3_access_key_id = your-access-key-id-from-the-downloaded-file
s3_secret_access_key = your-secret-access-key-from-the-downloaded-file
s3_bucket = your-s3-bucket-name
s3_base_dir = directory-where-backups-will-be-stored-in
```

bajo la sección `[pgmoneta]`.

## Garage Tutorial

Si Garage ya está descargado y configurado con una clave de acceso, clave secreta y bucket de S3, el flujo es:

1. Inicia Garage.
2. Configura pgmoneta para el endpoint de S3 de Garage.
3. Inicia pgmoneta.
4. Ejecuta `pgmoneta-cli s3 ls primary`.

Ejemplo de entradas de `pgmoneta.conf` bajo `[pgmoneta]`:

``` ini
storage_engine = s3
s3_access_key_id = your-garage-access-key-id
s3_secret_access_key = your-garage-secret-access-key
s3_bucket = your-garage-bucket
s3_base_dir = pgmoneta
s3_endpoint = your-garage-endpoint-host
s3_port = 3900
s3_region = garage
s3_use_tls = off
```

Inicia pgmoneta:

``` sh
pgmoneta -c pgmoneta.conf -u pgmoneta_users.conf
```

Lista objetos remotos:

``` sh
pgmoneta-cli s3 ls primary
```

`pgmoneta-cli s3 ls` lista objetos recursivamente en la ruta remota configurada e imprime todos los objetos descubiertos en una salida.

Elimina objetos remotos por prefijo (prefix):

``` sh
pgmoneta-cli s3 delete primary <prefix>
```

`pgmoneta-cli s3 delete` elimina todos los objetos en `<s3_base_dir>/<server>/backup/<prefix>`.

Restaurar un backup desde S3:

``` sh
pgmoneta-cli s3 restore primary <label>
```

`pgmoneta-cli s3 restore` descarga todos los archivos desde S3 para la etiqueta de backup indicada al directorio de backup local.

Los archivos de metadatos (`backup.info`, `backup.sha512`, `backup.manifest`) se descargan primero como archivos temporales y luego se renombran en orden después de que todos los archivos de datos son restaurados. `backup.info` se renombra al último y su presencia indica una restauración completa.
